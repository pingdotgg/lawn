## Cursor Cloud specific instructions

### Overview

lawn is a video review platform for creative teams. It uses:

- **Frontend**: React 19 + TanStack Router/Start + Vite 7 + Tailwind CSS 4
- **Backend**: Convex (BaaS — serverless functions + real-time DB)
- **Auth**: Clerk
- **Video**: Mux (transcoding/streaming)
- **Storage**: Railway Object Storage (S3-compatible)
- **Payments**: Stripe (partially implemented)

### Running the app

Standard dev commands are in `package.json`:

- `bun run dev` — starts both Vite dev server (port 5296) and Convex dev backend concurrently
- `bun run dev:web` — starts only the Vite frontend dev server (port 5296)
- `bun run lint` — ESLint across `app`, `src`, and `convex` directories
- `bun run typecheck` — TypeScript type checking (`tsc --noEmit`)
- `bun run typecheck:convex` — Convex-specific type checking

### Environment variables

The frontend requires these Vite env vars to render (checked at module load time):

- `VITE_CONVEX_URL` — Convex deployment URL (set automatically when running `bunx convex dev`)
- `VITE_CLERK_PUBLISHABLE_KEY` — Clerk publishable key (available in secrets)

To start only the frontend without Convex, set `VITE_CONVEX_URL` to a placeholder value:
```
VITE_CONVEX_URL="https://placeholder.convex.cloud" bun run dev:web
```

### Gotchas

- Bun v1.3.6 is the required package manager (specified in `package.json` `packageManager` field). It must be installed separately — the VM only has Node.js by default.
- The `bun run dev` command requires Convex authentication (`bunx convex dev` prompts for login). For frontend-only work, use `bun run dev:web` with a placeholder `VITE_CONVEX_URL`.
- The app's `__root.tsx` wraps all routes in `ClerkProvider` and `ConvexClientProvider`, so both `VITE_CLERK_PUBLISHABLE_KEY` and `VITE_CONVEX_URL` must be set for any page to render.
- The ESLint config lives in `eslint.config.mjs` (flat config format, ESLint 9). There are pre-existing unused-variable warnings in dashboard route files.
- TanStack Router auto-generates `app/routeTree.gen.ts` — do not edit this file manually.
